# m0.4 PMO 运维 (DEC-2026-0002)

**PMO 运维 v0.4.0** — 监控 + 告警 3 层 + 恢复 + 灾备 + Token 预算

## 核心模块

`scripts/runtime/operations/operations.py` — PMO 运维监控器 (m0.4)

## 6 大运维能力

### 1. 监控 (Monitor)

| 监控对象 | 维度 | 谁监控 |
|---|---|---|
| **业务项目状态** | 维度 1 | L1 PMO-Main |
| **业务项目 5 项基础指标** | 维度 3 | L2 PMO-Monitor-Agent |
| **业务项目 Token 预算** | 维度 1 | L1 PMO-Main |
| **业务项目内研发 5 阶段** | 维度 2 | L2 PMO-Engineer-Agent |

### 2. 告警 3 层 (0.0.10, DEC-2026-0002 边界清晰化)

| 层级 | 含义 | 触发条件 | 处理方 |
|---|---|---|---|
| **L1 业务自给** (biz-self) | 业务内部告警 | 业务自处理 | 业务项目 |
| **L2 PMO 实例** (pmo) | 项目级告警 | 业务升级 → PMO | PMO 实例 |
| **L3 Sponsor** (sponsor) | PMO 重大异常 | PMO 升级 → Sponsor | Sponsor |

### 3. 3 层异常拦截 (DEC-2026-0002 边界清晰化)

| 层 | 谁拦截 | 拦截什么 |
|---|---|---|
| **L0 PMO 规范** | PMO 规范 (不参与业务) | 标准+监督, 不拦截 |
| **L1 PMO 实例** | L1 PMO-Main + L2 PMO-Engineer-Agent + L2 PMO-Monitor-Agent | 研发异常 + 项目异常 |
| **L2 业务项目** | 业务项目自管 | 业务异常 (业务逻辑/业务数据/业务合规) |

### 4. 恢复 (Recovery)

6 种恢复策略:
- **RETRY** (重试)
- **FALLBACK** (降级)
- **RESTART** (重启)
- **ROLLBACK** (回滚)
- **ESCALATE** (升级)
- **SKIP** (跳过)

### 5. 灾备 (Disaster Recovery)

业务项目备份 → `archive/disaster-backups/<project-id>/<backup-id>`

每个备份包含:
- backup_id
- project_id
- source_path
- backup_path
- timestamp

### 6. Token 预算按业务项目分 (4 维)

每个业务项目 4 维 Token 预算:
- **Token** (业务流 token 消耗)
- **时间** (业务流耗时)
- **存储** (业务数据存储)
- **并发** (业务流并发)

Token 预算告警阈值:
- **80%** 警告 (warning)
- **100%** 严重 (critical)

## 关键 API

```python
from operations.operations import OperationsMonitor, AlertLevel, AlertSeverity, RecoveryStrategy

op = OperationsMonitor("/path/to/pmo")

# 1. 监控业务项目状态
op.monitor_project_state("1.1", "active")

# 2. 监控 5 项基础指标
op.monitor_metrics_5("1.1", {"flow_latency": 100, ...})

# 3. 监控 Token 预算
op.monitor_token_budget("1.1", used=10000, quota=100000)

# 4. 触发告警 (3 层)
op.trigger_alert(AlertLevel.PMO_INSTANCE, AlertSeverity.WARNING, "CODE001", "message", "1.1")

# 5. 执行恢复 (6 策略)
op.execute_recovery("1.1", RecoveryStrategy.RETRY, "原因")

# 6. 灾备
op.disaster_backup("1.1", "biz-projects/1.1-pmo-self/")

# 7. 获取运维报告
report = op.get_operations_report()
```
