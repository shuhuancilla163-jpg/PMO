# m0.6 上下文压缩 (Context Compression)

**上下文压缩 v0.1.0** — Token 经济 + 上下文管理 + Memory 分级联动

## 核心原则

上下文压缩是 PMO 工程化（0.0.2）的关键手段，确保 LLM context 不超限、Token 消耗可控。

> **0.0.4 失效检测**: Token 暴涨/上下文超限 → 强化上下文压缩

## 触发条件

### 2 级触发

| 级别 | 触发阈值 | 动作 |
|---|---|---|
| **WARNING** | context 使用达到 70% | 预警，记录日志，不强制压缩 |
| **CRITICAL** | context 使用达到 85% | 立即执行上下文压缩 |
| **BLOCKED** | context 使用达到 95% | 停止接收新任务，执行深度压缩，直到降至 70% 以下 |

### Token 预算（按业务项目）

每个业务项目的 4 维 Token 预算：

| 维度 | 描述 | 告警阈值 |
|---|---|---|
| **日 Token 消耗** | 每日 Token 上限 | 80% 警告 / 100% 严重 |
| **单次上下文 Token** | 单次 LLM 调用上限 | 70% 警告 / 85% 强制压缩 |
| **累计 Token 存储** | 历史 Token 存储配额 | 80% 警告 / 100% 归档 |

## 压缩策略（3 层）

### 第 1 层：热数据压缩（当前活跃）

**目标**：保留核心任务状态，压缩中间过程

| 内容类型 | 保留策略 | 压缩方式 |
|---|---|---|
| 当前任务上下文 | 完整保留 | 不压缩 |
| 当前对话历史 | 保留最近 3 轮 | 折叠更早轮次 |
| 中间推理结果 | 只保留最终结论 | 删除过程细节 |
| 临时变量/中间态 | 归档至 warm | 标记并移出 |
| 日志 trace | 保留摘要 | 删除详细 trace |

**压缩公式**：

```
保留 = 当前任务核心 + 最近3轮对话 + 最终结论 + 日志摘要
删除 = 中间推理过程 + 临时变量 + 详细trace
```

### 第 2 层：温数据压缩（近期活跃）

**目标**：提取关键摘要，压缩为引用格式

| 内容类型 | 保留策略 | 压缩方式 |
|---|---|---|
| 近期决策 | 保留决策 ID + 结论 | 删除讨论过程 |
| 近期任务结果 | 保留结果 + 状态 | 删除执行细节 |
| 近期指标数据 | 保留统计摘要 | 删除原始数据点 |
| 近期异常记录 | 保留类型 + 处理结果 | 删除堆栈详情 |

**压缩公式**：

```
保留 = 决策ID + 任务状态 + 指标摘要 + 异常类型
删除 = 讨论过程 + 执行细节 + 原始数据 + 详细堆栈
```

### 第 3 层：冷数据归档（历史）

**目标**：完全移出上下文，按需检索

| 内容类型 | 保留策略 | 存储位置 |
|---|---|---|
| 历史决策全文 | 不可变存储 | `immutable/3-decisions/` |
| 历史任务记录 | 不可变存储 | `archive/tasks/` |
| 历史指标原始数据 | 统计聚合后归档 | `metrics/archive/` |
| 复盘报告 | 完整保留 | `memory/cold/retrospectives/` |
| 阶段验收记录 | 完整保留 | `memory/cold/gate-reviews/` |

## 不可压缩内容

以下内容在任何情况下都不得从上下文压缩中删除或折叠：

| 内容类型 | 不可压缩原因 |
|---|---|
| **决策日志 ID** | 决策日志为不可变记录（0.0.1），是审计追溯的唯一依据 |
| **不可变文档 Hash** | Hash 是文档完整性的验证机制，删除将导致不可验证 |
| **阶段门控状态** | 门控状态是流程合规的核心，丢失将导致流程失效 |
| **业务项目注册信息** | E1 元数据是业务项目身份，PMO 监管的基础 |
| **异常拦截规则** | 3 层告警机制的基础配置，不可丢失 |
| **当前正在执行的任务** | 正在运行的任务上下文是业务连续性的保证 |

## 与 Memory 分级联动

```
[上下文压缩] ──→ [Memory 分级]
     │
     ├─ 热数据 (hot) ←── 压缩后保留在 context
     │      内容: 当前任务 + 最近3轮 + 结论 + 摘要
     │
     ├─ 温数据 (warm) ←── 从 hot 归档 + 从 cold 提取
     │      触发: 任务完成 / 热数据达到阈值
     │      内容: 近期决策 + 任务结果 + 指标摘要
     │
     └─ 冷数据 (cold) ←── 从 warm 归档
            触发: 阶段完成 / 手动归档
            内容: 历史决策 + 原始指标 + 复盘报告
```

### 归档规则

| 流转方向 | 触发时机 | 操作 |
|---|---|---|
| **hot → warm** | 任务完成时 | 热数据归档到温，保留摘要 |
| **warm → cold** | 阶段完成时 | 温数据归档到冷，聚合统计 |
| **cold → 不可变** | 重大决策时 | 冷数据中的决策写入 immutable |

## 上下文压缩执行流程

```
1. [检测] Token/上下文监控器检测到阈值
   ↓
2. [评估] 判断压缩级别 (WARNING / CRITICAL / BLOCKED)
   ↓
3. [选择策略] 根据数据层级选择压缩策略
   ↓
4. [执行压缩] 按层执行压缩（不可压缩内容跳过）
   ↓
5. [归档] 压缩数据按规则归档至 hot/warm/cold
   ↓
6. [验证] 验证压缩后上下文大小 < 阈值
   ↓
7. [记录] 记录压缩日志 (memory/hot/compression-log/)
```

## 自检（D15 上下文管理）

m1.5 自检新增 **D15 上下文管理自检**：

| 检查项 | 检查内容 | 失败表现 |
|---|---|---|
| 压缩触发正常 | WARNING 70% / CRITICAL 85% / BLOCKED 95% 正确触发 | 超限未触发 |
| 不可压缩内容保留 | 决策ID/Hash/门控状态/注册信息不丢失 | 关键内容被删 |
| Memory 分级联动 | hot→warm→cold 流转正确 | 分级混乱 |
| Token 预算不超 | 日 Token / 单次 Token 不超标 | 预算超限 |

## 与 0.0.4 失效检测对应

| 失效表现 | 修复手段 |
|---|---|
| Token 暴涨 | 上下文压缩 CRITICAL 级立即执行 |
| 上下文超限 | BLOCKED 级深度压缩 + 新任务暂停 |
| 热数据堆积 | 任务完成时强制 hot→warm 归档 |
| 温数据过期 | 阶段完成时强制 warm→cold 归档 |

## 关键约束

1. **不可压缩内容绝对保留** — 决策日志、Hash、门控状态永远不删
2. **压缩可逆** — 所有压缩数据归档到 Memory 分级，按需可检索
3. **压缩有日志** — 每次压缩记录操作日志，支持审计
4. **Token 预算硬限制** — 100% 时禁止新的 LLM 调用，直到压缩完成

## 关键决策

- **m0.6 v0.1.0**: 上下文压缩规范 (2026-06-19)
- **3 层触发**: WARNING 70% / CRITICAL 85% / BLOCKED 95%
- **3 层压缩**: 热数据细节压缩 / 温数据摘要提取 / 冷数据完全归档
- **不可压缩内容**: 决策日志 + Hash + 门控状态 + 注册信息 + 告警规则
- **Memory 联动**: hot→warm→cold 按规则流转
- **自检 D15**: 上下文管理自检验证压缩正确性
