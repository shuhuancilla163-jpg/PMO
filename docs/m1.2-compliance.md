# m1.2 PMO 合规模块 (DEC-2026-0002)

**PMO 合规 v0.5.0** — 5 项基础合规工具 + DEC-2026-0002 加 3 项

## 核心模块

`scripts/runtime/compliance/compliance.py` — PMO 合规检查器 (m1.2 + DEC-2026-0002)

## 5 项基础合规工具 (m1.2)

### C11: gitleaks 密钥扫描
- 扫描代码中常见密钥模式
- 检测: AWS Access Key / GitHub Token / OpenAI API Key / Private Key / Generic API Key
- 状态: pass / fail

### C12: redact.py 脱敏
- 脱敏规则: email / phone / id_card / credit_card
- 输出: 脱敏后文本 + 规则命中统计

### C13: 数据分级检查
- 4 级: public / internal / confidential / secret
- 自动判断 (看 key 名字)

### B13: redaction-rules 脱敏规则文档
- 检查规则文档存在性
- 占位: `immutable/0-governance/0.0.7-decoupling-4layers.md`

### B14: audit-log-spec 审计日志规范
- 检查决策日志目录
- 输出: 日志文件数

## DEC-2026-0002 加 3 项

### 业务项目考核合规
- 检查 Assessor-Agent 3 维度考核结果
- 状态: pass (verdict=compliant) / warning

### 监控合规
- 检查 3 维度监控 + 3 告警层
- 状态: pass / fail

### 消息流通合规
- 检查 Message-Broker-Agent 中介
- 状态: pass / fail

## 指标可贯彻验证 (m1.2)

每个指标必须 3 件事:
- **可采集** (可以拿到值)
- **可审计** (历史可查)
- **可贯彻** (有决策/动作)

检查:
- has_definition (schema.json)
- has_history (perf-baseline.json)
- has_decision_log (decisions/active/)
