## 6 类消息类型 (强制)

```
request        # 请求 (业务项目 A → B, 等响应)
response       # 响应 (B → A)
notification   # 通知 (单向, 异步)
alert          # 告警 (单向, 异步)
escalation     # 升级 (单向, 异步, 上行)
biz_event      # 业务事件
biz_data       # 业务数据
```

## 6 类主题 (强制命名规范)

```
biz.{biz_project_id}.state           # 业务项目状态变化
biz.{biz_project_id}.metric          # 业务项目指标上报
biz.{biz_project_id}.exception       # 业务项目异常
biz.{biz_project_id}.data            # 业务项目数据
inter.biz.{a}.to.{b}                # 业务项目间定向消息
pmo.{layer}.{action}                # PMO 内部消息 (L0/L1/L2)
```

## 审计日志 (append-only)

- 路径: `logs/message-broker/audit-YYYYMMDD.log`
- 按日滚动
- 不可删 (Git 跟踪)
- 字段: audit_id / action / from / to / topic / status / note / timestamp
- 动作: subscribe / unsubscribe / publish / deliver / ack / retry

## 强制字段 (协议校验)

每条消息必填:

```
msg_id, msg_type, from, to, topic, qos, timestamp, layer, content
```

## 业务项目接入 (5 步流程第 4 步)

业务项目接入 PMO 实例时:

1. 业务项目**注册**时声明订阅/发布主题 (`messaging.yaml`)
2. 业务项目通过 PMO 实例 API 发送/接收消息
3. 业务项目消息必须符合 PMO 协议
4. PMO 实例监管业务项目消息 (可监控 + 可审计)
5. 项目间消息进 PMO 指标看板 (Sponsor 可看)

## API 使用示例

```python
from scripts.runtime.protocol.message_broker import MessageBroker, MessageType, QoSLevel

broker = MessageBroker("/path/to/pmo")

# 1. 业务项目订阅
broker.subscribe("1.3", "inter.biz.1.2.to.1.3")

# 2. 业务项目发布 (经 broker)
msg = broker.publish(
    from_project="1.2", to_project="1.3",
    topic="inter.biz.1.2.to.1.3",
    msg_type=MessageType.NOTIFICATION,
    content={"event": "data_ready"},
    qos=QoSLevel.AT_LEAST_ONCE
)

# 3. 监控指标
metrics = broker.get_monitoring_metrics()
# {"stats": {11 项}, "topic_heat_top5": [...], ...}

# 4. 审计日志
audit = broker.get_audit_log(limit=10)
# [{"action": "publish", "from": "1.2", "to": "1.3", ...}, ...]
```
