## 核心

业务项目接入 PMO 实例时, 必须提交 3 项元数据, 全部由 PMO 存, 业务项目可查:

- **E1 业务项目元数据** (register.yaml): 业务项目身份信息
- **E2 业务数据 schema** (data-schema.yaml): 业务项目数据模型
- **E3 业务术语表** (glossary.yaml): 业务项目专业术语 + 业务 agent 身份

## E2 业务数据 schema (data-schema.yaml)

**路径**: `biz-projects/<biz_project_id>/data-schema.yaml`
**PMO 存**: `config/biz-meta/<biz_project_id>-schema.json`

业务项目**自定**业务数据 schema, PMO 存 + 验证, 业务流消息路由时使用。

**1.1-pmo-self E2 (示例)**:
- 实体: task (7 字段) / decision (5 字段) / metric (5 字段)
- 事件: task_started / task_completed / decision_made / metric_recorded

**1.2-finance E2 (示例)**:
- 实体: order (8 字段) / trade (5 字段) / position (6 字段) / risk_metric (6 字段)
- 事件: order_placed / order_filled / position_updated / risk_breached

**PMO 验证规则**:
- 实体名唯一
- primary_key 字段在 fields 内
- foreign_key 引用必须存在
- enum.values 必填
- 业务项目可任意扩展, PMO 不限制业务实体类型

## E3 业务术语表 (glossary.yaml)

**路径**: `biz-projects/<biz_project_id>/glossary.yaml`
**PMO 存**: `config/biz-meta/<biz_project_id>-glossary.json`

业务项目**自定**业务术语, 业务 agent 用术语, PMO 验证术语一致性。

**1.1-pmo-self E3 (示例)**:
- 术语: task / decision / phase / metric / agent / compliance / audit (7 个)
- 业务 agent role: Plan-Curator / Decision-Logger / Metric-Keeper / Compliance-Watcher / Self-Evolution-Agent (5 个)

**1.2-finance E3 (示例)**:
- 术语: order / trade / position / P&L / drawdown / VaR / signal / compliance_check (8 个)
- 业务 agent role: Data-Engineer / Quant-Analyst / Risk-Manager / Portfolio-Manager / Compliance-Officer / Reporting-Analyst (6 个)

**PMO 验证规则**:
- 业务 agent role 不与 5 阶段研发 role 重名 (DEC-2026-0003)
- 术语 category 必填 (entity/state/metric/action/role)
- 业务项目可任意扩展术语, PMO 不限制

## 5 阶段研发 agent role (边界检查, DEC-2026-0003)

```
Requirement-Engineer      # 5 阶段研发
Development-Engineer
Test-Engineer
Operations-Engineer
Evaluation-Engineer
```

业务 agent role 不能与上面 5 个重名。
