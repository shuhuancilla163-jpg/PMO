# m2.2 业务不可变文档 2 项 (DEC-2026-0006, v0.13.0)

## 1. 概述

**业务不可变文档** = 业务项目的业务产出文档, 与 PMO 治理不可变文档是两个独立层级:

| 层级 | 位置 | 管理方 | 内容 |
|---|---|---|---|
| PMO 不可变文档 | `immutable/0-governance/` | PMO | 治理规范 |
| **业务不可变文档** | `biz-projects/<id>/biz-docs/` | 业务项目 | 业务产出文档 |

## 2. 验收 2 项 (F1/F5)

| 检查项 | 路径 | 状态 |

- 业务设计文档 (`biz-design.md`): 基于需求文档
- 业务流文档 (`biz-flow.md`): 编排业务 agent 之间的消息流
- 发布说明 (`release-notes.md`): 按 semver 版本记录变更

### F5: 业务版本管理

**验收标准**: Git tag/release + semver 可演示

| 检查项 | 状态 |
|---|---|
| BizVersionStore 可实例化 | ✅ |
| release() → v1.0.0 | ✅ |
| release() → v1.0.1 | ✅ |
| next_minor() → v1.1.0 | ✅ |
| next_major() → v2.0.0 | ✅ |
| compare_versions() 正确 | ✅ |
| 版本存储文件生成 | ✅ |
| 版本历史正确 (4 条) | ✅ |

**semver 规则**:

```
v<MAJOR>.<MINOR>.<PATCH>

- MAJOR: 破坏性变更 (E2/E3 破坏性修改)
- MINOR: 新增功能 (E2 新增 entity, E3 新增 term)
- PATCH: 修复 / 文档更新
```

**Git tag 格式**: `biz.<biz_id>.v<MAJOR>.<MINOR>.<PATCH>`

## 3. 两层不可变文档分离

```
PMO 不可变文档 (immutable/0-governance/)
└── PMO 治理规范
    ├── 0.0.1 ~ 0.0.17 元规则
    ├── 决策日志 (decisions/)
    └── PMO 版本 (Git tag: pmo.vX.Y.Z)

业务不可变文档 (biz-projects/<id>/biz-docs/)
└── 业务产出文档
    ├── 01-requirements/  (需求)
    ├── 02-design/       (设计)
    ├── 03-implementation/ (实施)
    └── 04-release/      (发布, Git tag: biz.<id>.vX.Y.Z)
```

## 4. biz-docs 与 5 阶段的关系

| 研发阶段 | biz-docs 目录 | 主要产出文档 |
|---|---|---|
| requirement | 01-requirements/ | biz-requirements.md |
| development | 02-design/ + 03-implementation/ | biz-design.md + biz-flow.md |
| test | (测试报告在 reports/) | - |
| operations | 04-release/ | release-notes.md + deployment.md |
| evaluation | 版本评审 | 版本号确定 + Git tag |

## 5. 与 m2.1 的关系

- **E2 data-schema** → 业务设计文档的基础
- **E3 glossary** → 需求文档的基础
- m2.1 接入后 → 业务项目已有 E1/E2/E3 → 可开始写 biz-docs/
