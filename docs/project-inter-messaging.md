# 项目间消息流通机制 (DEC-2026-0002)

**业务项目↔业务项目消息经 PMO 实例中介** (PMO-Message-Broker-Agent)

## 核心

**业务项目间不直接通信**, 都通过 PMO 实例中介 (类似消息总线):
- 业务项目 = "分公司"
- PMO 实例 = "总公司的邮件系统"
- 业务项目间消息 = "分公司间的邮件, 都通过邮件系统"

**关键**:
- 业务项目不直接发消息 (避免耦合)
- PMO 实例管消息路由 (类似消息总线)
- 业务项目订阅 PMO 实例的消息主题

## 消息路由

| 路由类型 | 路径 |
|---|---|
| **业务项目↔业务项目** | 1.2 → PMO-Message-Broker → 1.3 (经 PMO 中介) |
| **业务项目→PMO 实例** | 业务项目 → PMO 实例 (直接) |
| **PMO 实例→业务项目** | PMO 实例 → 业务项目 (直接) |
| **PMO 实例→Sponsor** | PMO 实例 → Sponsor (直接, 通过 Sponsor 通知) |

## 消息类型

| 类型 | 含义 | 同步/异步 |
|---|---|---|
| **request/response** | 请求/响应 | 同步 |
| **notification** | 通知 | 异步 |
| **alert** | 告警 | 异步 |
| **escalation** | 升级 | 异步 |
| **biz_event** | 业务事件 (业务项目状态变化/业务指标超阈值/业务异常) | 异步 |
| **biz_data** | 业务数据 (业务结果/业务报告) | 异步 |

## 消息主题 (业务项目订阅)

| 主题 | 含义 |
|---|---|
| `biz.{biz_project_id}.state` | 业务项目状态变化 |
| `biz.{biz_project_id}.metric` | 业务项目指标上报 |
| `biz.{biz_project_id}.exception` | 业务项目异常 |
| `biz.{biz_project_id}.data` | 业务项目数据 |
| `inter.biz.{biz_project_id_a}.to.{biz_project_id_b}` | 业务项目间定向消息 |

**业务项目订阅**:
- 业务项目注册时声明订阅/发布主题
- 业务项目通过 PMO 实例 API 订阅
- 业务项目只能订阅自己有权的主题

## 消息协议

| 维度 | 规范 |
|---|---|
| **协议** | PMO 通信协议 (m0.2 protocol.py 扩展) |
| **格式** | JSON |
| **QoS** | at-least-once (至少一次) |
| **可监控** | 数量/成功率/延迟/失败重试 |
| **可审计** | 决策日志 (append-only) |

## 业务项目接入消息机制 (5 步流程的第 4 步)

业务项目接入 PMO 实例时:

1. **业务项目注册** 时声明消息订阅/发布主题
2. **业务项目** 通过 PMO 实例 API 发送/接收消息
3. **业务项目消息** 必须符合 PMO 协议
4. **PMO 实例** 监管业务项目消息 (可监控 + 可审计)
5. **项目间消息** 进 PMO 指标看板 (Sponsor 可看)

## 消息监控 (L2 PMO-Message-Broker-Agent)

| 监控项 | 监控方 |
|---|---|
| 消息数量 | L2 PMO-Message-Broker-Agent |
| 消息成功率 | L2 PMO-Message-Broker-Agent |
| 消息延迟 | L2 PMO-Message-Broker-Agent |
| 消息失败重试 | L2 PMO-Message-Broker-Agent |

## 关键: 业务项目间消息经 PMO 中介的好处

| 好处 | 说明 |
|---|---|
| **避免耦合** | 业务项目不直接通信, 通过 PMO 中介 |
| **可监控** | PMO 实例管所有项目间消息, 可监控 |
| **可审计** | PMO 实例存消息日志, 可审计 |
| **可路由** | PMO 实例管消息路由, 业务项目不需知道对方地址 |
| **可重试** | PMO 实例管消息失败重试 |
| **可扩展** | 加新业务项目时, PMO 实例自动接入消息机制 |

## 类比

- 业务项目 = "分公司"
- 业务项目间消息 = "分公司间的业务沟通"
- PMO 实例 = "总公司的邮件系统/内部通讯平台"
- 业务项目不直接发消息 (避免混乱), 都通过邮件系统
- 总公司监管所有分公司邮件 (可监控/可审计)

## 关键决策

- **DEC-2026-0002**: 14 块关键调整 (2026-06-18)
- **新增角色**: L2 PMO-Message-Broker-Agent (项目间消息)
- **影响**: m0.2 通信协议扩展, m2.5 跨边界契约加项目间消息契约
