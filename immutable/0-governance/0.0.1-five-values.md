# 0.0.1 五大核心价值 (Sponsor 视角)

**不可变原则, 不变 (0.0.8 约束)**

## 5 价值

| # | 价值 | 含义 | 落地手段 |
|---|---|---|---|
| 1 | **高效流程** | 减少冗余, 一气呵成 | 阶段门控 + 并行化 + 模板化 + 触点抽象 |
| 2 | **规范** | 一致性, 可预测 | 不可变文档 + 模板 + 接口契约 + 角色契约 |
| 3 | **可控** | 状态可见, 风险可拦 | 阶段门控 + 自检 + 性能管理 + 异常拦截 |
| 4 | **验收** | Sponsor 拍板, 责任清晰 | 阶段验收 + MVP 验收 + 决策日志 + 不可变文档 |
| 5 | **优化** | 持续改进, 自进化 | 性能管理 + 反思机制 + 规则演进 + Memory Manager |

## 价值失效 → 检测 → 修复

| 价值失效 | 表现 | 修复 |
|---|---|---|
| 高效流程失效 | 流程卡住, 反复重做 | 重新设计阶段门控 |
| 规范失效 | agent 行为不一致, 输出格式乱 | 强化不可变文档 + 模板 |
| 可控失效 | 不知道当前状态, 风险不可见 | 加强自检 + 状态报告 |
| 验收失效 | Sponsor 不清楚拍板时机 | 强化阶段验收节点 |
| 优化失效 | 重复犯错, 规则不演进 | 强化 Memory Manager + 反思 |
