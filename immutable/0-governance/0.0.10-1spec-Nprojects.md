# 0.0.10 PMO 1 套规范 + N 项目复用原则 (Sponsor 平等讨论后修订)

**不可变 (0.0.10, Sponsor 平等讨论后)**

## 核心

**1 套 PMO 治理规范, 1 个 PMO 实例, N 个业务项目复用, 1 个 Sponsor 介入**。

## 关键澄清 (Sponsor 平等讨论后, 重要)

| 维度 | 我之前理解 (偏了) | Sponsor 理解 (正确) |
|---|---|---|
| **PMO 数量** | 1 个总 PMO + N 个项目 PMO | **1 套 PMO 规范, 1 个 PMO 实例, 管 N 项目** |
| **业务项目** | 1 个 PMO 项目 + N 个业务项目 | 1.1 PMO 自建 + 1.2 ~ 1.N 业务项目 |
| **每项目有 PMO** | 是 (项目 PMO) | **否 (PMO 是规范, 1 个 PMO 实例管所有项目)** |
| **治理重复** | 高 (总 PMO + 项目 PMO 治理规则重复) | **低 (1 套规范, 1 个实例)** |
| **复杂度** | +10 | **+0** (不增加复杂度) |
| **基座完善** | 体现 (5 层) | **更彻底 (1 套规范, N 项目复用)** |

## PMO 规范 (1 套, 不变)

- 5 价值 + 1 工程化 (0.0.1, 0.0.2)
- 0.0.7 价值-治理-工程-项目实现 解耦 (4 层)
- 0.0.8 自进化原则 (PMO 规范自进化, 不区分 PMO / 项目 PMO)
- 0.0.9 第一阶段基座完善原则
- 三权分立 (L0 监督, L1 行政, L2 司法)

## PMO 实例 (1 个, 不变)

- 1 个 PMO 实例管 N 个业务项目
- 业务项目注册到 PMO 实例 (1 套契约, m2.5)
- 业务项目状态机由 PMO 实例管 (m2.6)
- 业务项目资源配额由 PMO 实例管 (m2.6)
- 业务项目归档由 PMO 实例管 (m2.6)
- 业务项目监控由 PMO 实例管 (m2.6)
- 业务项目 checklist 由 PMO 实例管 (m2.6)

## Sponsor (1 个, 不变)

- 1 个 Sponsor (你) 介入所有项目
- 多项目时: Sponsor 接入所有项目, 同一规范
- 监督: Sponsor 看 PMO 实例的指标看板
- 介入节点: 1.1 启动, 1.2 启动, 1.3/1.4 启动, 重大决策

## 业务项目 (N 个, 可变)

| 项目 | 阶段 | 说明 |
|---|---|---|
| **1.1 PMO 系统自建** | 第一阶段 (当前) | PMO 平台搭建 + 1 套规范 + 1 个实例 + 自测 |
| **1.2 业务系统** | 第一阶段 (1.1 完成后) | 第 1 个业务项目, 用 1 套 PMO 规范接入 |
| **1.3 / 1.4 / ... 业务系统** | 后续 (1.2 完成后) | 第 2/3/N 个业务项目, 用同一套 PMO 规范接入 |

- 业务配额 (m2.6)

## 三权分立

| 层级 | 名称 | 职责 |
|---|---|---|
| **L0** | Sponsor 监督权 | 顶层权威, 打破递归, 看指标看板, 不执行 |
| **L1** | PMO 行政权 | 1 个 PMO 实例, 管 N 项目, 执行自治, 出指标 |
| **L2** | PMO 司法权 | 3 个 L2 核心 agent 互相制衡, 审计 L1, 出指标, 互相验证 |

## 3 层异常拦截 (Sponsor 平等讨论后, 重要)

| 层 | 谁拦截 | 拦截什么 |
|---|---|---|
| **PMO 规范** | 不参与业务 | 标准制定 + 监督 |
| **PMO 实例** | 拦截项目级 (含 PMO 自身) | 项目边界 (注册/状态/配额/归档) |
| **业务项目** | 拦截自身 | 业务内部 (业务异常/性能/合规) |

**关键**:
- PMO 规范不参与业务 = 治理层不干预业务执行
- PMO 实例拦截项目级 = 治理层管项目边界
- 业务项目拦截自身 = 业务自治

## 3 层告警 (Sponsor 反问后, 重要)

| 层 | 谁告警 | 谁接收 | 处理 |
|---|---|---|---|
| **业务内部** | 业务 agent | 业务项目自身 | 业务项目处理 |
| **项目级 (含 PMO 自身)** | PMO 实例 | PMO 实例自身 | PMO 实例处理 |
| **PMO 重大** | PMO 实例 | Sponsor | Sponsor 决策 |

**关键**: 业务告警业务自给, 自己处理, 重大告警才→PMO→Sponsor。

## 指标可贯彻 (Sponsor 点出后, 重要)

- 业务指标 5 项 (流耗时/异常率/通过率/回滚率/Token) - 业务项目可加自定义
- 治理指标 (阶段门控通过率/决策日志完整度/自检覆盖率/自进化次数)
- 工程指标 (Token 消耗/部署成功率/性能/可用性/异常拦截率)

**关键**: 指标要**跑出来** (可贯彻), 不是"应该做"。Sponsor 看指标看板, 不只看 PMO 报告。

## 关键

- 1.1 完成后, 1.2/1.3/1.4 都**不用改 PMO 规范**, 只需要注册到 PMO 实例即可
- 1.2 完成后, 1.3/1.4 都**不用改业务接入路径**, 只需要复用 m2.5 + m2.6
- **基座完善原则 (0.0.9) 彻底体现**: 1 套规范 + 1 套接入路径, N 项目复用

## 不可变性

- 1 套规范 + 1 个实例 + 1 个 Sponsor**不可变**
- 3 层异常拦截 + 3 层告警 + 3 权分立**不可变**
- 任何变更需 P9 决策 + Sponsor 拍板
