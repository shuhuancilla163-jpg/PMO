# 0.0.13 业务项目内研发 5 阶段 agent 输入输出规范 (DEC-2026-0003)

**5 阶段 agent 是业务项目内实施的 agent, PMO 只定输入输出规范 + 监督, 不实施**

## 核心原则

**5 阶段 agent 物理位置**: 业务项目内 (`biz-projects/<biz_project_id>/eng-roles/`)
**PMO 职责**: 提供初始模板 + 输入输出规范 + 7 项合规监督
**业务项目职责**: 从 PMO 模板复制到 `eng-roles/`, 然后按业务调整实施
**PMO 不实施 5 阶段 agent**

## 5 阶段 agent 通用规范

### Agent 类 (业务项目自己实现)

| 阶段 | 必选 | 业务项目可调 | 实施 |
|---|---|---|---|
| 1. Requirement-Engineer | 否 (业务项目可选) | 可简化/跳过 | 业务项目 |
| 2. Development-Engineer | **是** (必选) | 可加业务特定子阶段 | 业务项目 |
| 3. Test-Engineer | **是** (必选) | 可加业务特定测试类型 | 业务项目 |
| 4. Operations-Engineer | **是** (必选) | 可加业务特定运维场景 | 业务项目 |
| 5. Evaluation-Engineer | 否 (业务项目可选) | 可简化/跳过 | 业务项目 |

**关键**: 必选阶段业务项目**必须实施**, 可选阶段业务项目可关闭。

## 业务项目可调 5 阶段 (但必须符合 PMO 7 项)

业务项目**可调** 5 阶段 agent:
- **简化**: 关闭需求阶段 + 评估阶段 (业务项目自定简化为 3 阶段: 研发/测试/运维)
- **扩展**: 业务项目可加业务特定子阶段 (如: 需求阶段加 "业务建模" 子阶段)
- **改名**: 业务项目可改 agent 名字 (如: Development-Engineer → Quant-Engineer)
- **调输入输出**: 业务项目可调输入输出字段 (但必须保留 PMO 7 项合规字段)
- **业务特定工具**: 业务项目可加业务特定工具 (如: 量化金融的策略回测工具)

**但必须**:
- 必选阶段 (研发/测试/运维) 不可关闭
- PMO 7 项合规检查项**不可减少** (可增加)
- 必选阶段必须有 PMO 7 项合规字段

## 业务项目 eng-roles/ 注册契约 (YAML)

业务项目接入 PMO 时, 必须在 `eng-roles/register.yaml` 声明:

```yaml
# biz-projects/<biz_project_id>/eng-roles/register.yaml

  biz_project_id: "1.2-finance"
  pmo_compliance_version: "0.0.13"  # PMO 规范版本
  template_source: "PMO/templates/eng-roles/"  # 模板来源
  
  stages:
    requirement:
      enabled: true
      agent_class: "eng_roles.01_requirement_engineer.RequirementEngineer"
      customization: "业务项目加业务建模子阶段"  # 调整说明
      pmo_7_compliance: 7  # 必须 7 项
      pm0_supervised: true
    
    development:
      enabled: true
      agent_class: "eng_roles.02_development_engineer.DevelopmentEngineer"
      customization: "业务项目用 Python + 量化库"
      pmo_7_compliance: 7
      pm0_supervised: true
    
    test:
      enabled: true
      agent_class: "eng_roles.03_test_engineer.TestEngineer"
      customization: "业务项目加策略回测测试"
      pmo_7_compliance: 7

    operations:
      enabled: true
```

## PMO Engineer-Agent 监督 (L2)

**PMO Engineer-Agent 监督内容** (维度 2):
- 业务项目 5 阶段**上报数据** (`report_to_pmo()` 返回)
- 业务项目 5 阶段**符合 PMO 7 项合规** (`pmo_7_compliance` 字段)
- 业务项目 5 阶段**决策日志**完整性
- 业务项目 5 阶段**不可变文档**数量
- 业务项目 5 阶段**异常拦截率**

**PMO Engineer-Agent 不监督**:
- 业务项目 5 阶段 agent **具体实现代码** (PMO 不读业务项目代码)
- 业务项目 5 阶段 agent **业务特定工具** (PMO 不干预业务内容)
- 业务项目 5 阶段 agent **业务特定输入输出字段** (PMO 只看 PMO 7 项合规字段)

## 5 阶段契约生效条件

业务项目 5 阶段契约生效:
1. 业务项目 `eng-roles/register.yaml` 已声明
2. 5 阶段 (或简化 2-3 阶段) 全部有 `agent_class` 指向具体实现
3. 必选阶段 (研发/测试/运维) 全部 `enabled: true`
4. 各阶段 `pmo_7_compliance` 全部 ≥ 7
5. 各阶段上报接口 `report_to_pmo()` 实现完整

## 5 阶段契约违反处理

| 违反 | 处理 |
|---|---|
| 必选阶段未实施 | critical + 业务项目暂停 |
| 阶段不符合 PMO 7 项 | 警告 + 业务项目整改 |
| 阶段未走 PMO 阶段门控 | critical + 业务项目暂停 |
| 阶段未记录决策日志 | 警告 + 补录决策日志 |
| 阶段不可变文档缺失 | 警告 + 补录不可变文档 |
| 阶段异常拦截失败 | critical + 升级到 Sponsor |
| 阶段指标未上报 | 警告 + 补报指标 |
| 阶段告警失败 | 告警通道修复 + 重新告警 |
| 阶段未给 Sponsor 报告 | 警告 + 补 Sponsor 报告 |
