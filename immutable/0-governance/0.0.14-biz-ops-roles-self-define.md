# 0.0.14 业务需求运营 agent 业务项目自定原则 (DEC-2026-0003)

**业务需求运营 agent 完全由业务项目自定, PMO 不预设不干预**

## 核心原则

**业务需求运营 agent** = 业务项目内, 业务项目自己定义业务相关的 agent, PMO 不预设不干预。

| 维度 | 5 阶段 agent (PMO 监管) | 业务需求运营 agent (业务自管) |
|---|---|---|
| 物理位置 | `biz-projects/<id>/eng-roles/` | `biz-projects/<id>/biz-agents/` |
| 实施 | 业务项目从 PMO 模板复制 + 调整 | 业务项目完全自定 |
| PMO 职责 | 给模板 + 定规范 + 7 项合规监督 | **不预设不干预** |
| 数量 | 5 个 (或简化 2-3 个) | 业务项目自定 (不固定 8) |
| 命名 | PMO 规范 (Requirement-Engineer 等) | 业务项目自定 (Data-Engineer 等) |
| 输入输出 | PMO 规范 (0.0.13) | 业务项目自定 |
| 元规则 | 0.0.11 + 0.0.13 | **0.0.14 (本规则)** |

## 业务需求运营 agent 业务项目自定

### 业务项目可自定 (PMO 不干预)

业务需求运营 agent 的**所有方面**由业务项目自定:

1. **业务场景**: 业务项目自选 (如: 量化金融/电商推荐/内容生成/工业制造)
2. **业务 agent 名字**: 业务项目自定 (如: Data-Engineer / Quant-Analyst / Risk-Manager)
3. **业务 agent 数量**: 业务项目按业务场景配, **不固定 8 个**
   - 简单业务项目: 2-3 个业务 agent
   - 中等业务项目: 4-6 个业务 agent
   - 复杂业务项目: 7-10 个业务 agent
4. **业务 agent 责任**: 业务项目自定
5. **业务 agent 输入输出**: 业务项目自定
6. **业务 agent 性格**: 业务项目自定
7. **业务 agent 工具**: 业务项目自定
8. **业务 agent 实现语言/框架**: 业务项目自定
9. **业务 agent 业务流程**: 业务项目自维护业务流模板
10. **业务 agent 异常定义**: 业务项目自定义具体业务异常
11. **业务 agent 业务指标**: 业务项目自维护业务全量数据

### PMO 不预设

PMO **不预设**:
- ❌ 业务场景模板
- ❌ 业务 agent 数量 (不固定 8)
- ❌ 业务 agent 名字
- ❌ 业务 agent 责任/输入输出
- ❌ 业务流程
- ❌ 业务异常
- ❌ 业务指标

### PMO 不干预

PMO **不干预**:
- ❌ 业务 agent 的具体实现代码
- ❌ 业务 agent 的业务逻辑
- ❌ 业务 agent 的业务数据
- ❌ 业务 agent 之间的协作
- ❌ 业务 agent 的异常处理 (业务异常由业务项目自管)
- ❌ 业务 agent 的指标定义 (业务指标由业务项目自管)

## 业务项目 biz-agents/ 目录约定

虽然 PMO 不预设, 但建议业务项目按以下目录组织 (业务项目可调):

```
biz-projects/<biz_project_id>/
├── biz-agents/                # 业务需求运营 agent (业务自定)
│   ├── 01-data-engineer.py    # 业务 agent 1
│   ├── 02-quant-analyst.py    # 业务 agent 2
│   ├── ...                    # 业务项目自定
│   └── README.md              # 业务 agent 清单
```

**业务 agent 命名建议** (业务项目可调):
- 业务项目可自定命名 (如: `data-engineer.py` / `risk-mgr.py` / `quant-v2.py`)
- 建议使用业务角色命名 (如: `data-engineer` / `risk-manager`)
- 业务项目可加版本号 (如: `quant-analyst-v2.py`)

**业务 agent 数量建议** (业务项目可调):
- 简单业务项目: 2-3 个 (如: 1 数据 + 1 分析 + 1 报告)
- 中等业务项目: 4-6 个 (如: 1 数据 + 1 策略 + 1 风控 + 1 组合 + 1 报告)
- 复杂业务项目: 7-10 个 (更多细分业务角色)

**关键**: 业务项目**完全自定**, PMO **不强制**任何规范。

## 业务项目 biz-agents 契约 (YAML, 业务项目自填)

业务项目接入 PMO 时, **建议**在 `biz-agents/register.yaml` 声明业务 agent 清单 (PMO 不强制):

```yaml
# biz-projects/<biz_project_id>/biz-agents/register.yaml
# 注意: 此文件 PMO 不强制要求, 业务项目自愿
biz_agents:
  biz_project_id: "1.2-finance"
  pmo_supervised: false  # 业务 agent PMO 不监管
  agent_count: 6  # 业务项目自定数量 (演示 6 个)
  
  agents:
    - name: "Data-Engineer"
      file: "01-data-engineer.py"
      responsibility: "金融数据采集/清洗/存储"
      business_scenario: "金融数据 ETL"
    
    - name: "Quant-Analyst"
      file: "02-quant-analyst.py"
      responsibility: "量化策略研究/回测"
      business_scenario: "量化策略"
    
    - name: "Risk-Manager"
      file: "03-risk-manager.py"
      responsibility: "风险监控/预警"
      business_scenario: "风控"
    
    - name: "Portfolio-Manager"
      file: "04-portfolio-mgr.py"
      responsibility: "组合管理/调仓"
      business_scenario: "组合管理"
    
    - name: "Compliance-Officer"
      file: "05-compliance-officer.py"
      responsibility: "合规检查/审计"
      business_scenario: "合规"
    
    - name: "Reporting-Analyst"
      file: "06-reporting-analyst.py"
      responsibility: "业务报告/可视化"
      business_scenario: "报告"
```

**说明**:
- 业务 agent 数量 6 个 (演示, 业务项目可自定其他数量)
- 业务 agent 名字 业务项目自定
- 业务 agent 责任 业务项目自定
- 业务 agent 业务场景 业务项目自定
- PMO **不**监督**任何字段** (业务自管)

## PMO 监督 vs 业务自管边界

| 维度 | 5 阶段 agent (PMO 监管) | 业务需求运营 agent (业务自管) |
|---|---|---|
| 实施位置 | 业务项目内 eng-roles/ | 业务项目内 biz-agents/ |
| 实施 | 业务项目从 PMO 模板复制 + 调整 | 业务项目完全自定 |
| 输入输出 | PMO 规范 (0.0.13) | 业务项目自定 |
| PMO 7 项合规 | 必选 (阶段门控/决策日志/不可变文档/异常拦截/指标可贯彻/3 层告警/Sponsor 报告) | **不要求** |
| PMO 监督 | 看上报数据是否合规 | **不监督** |
| PMO 干预 | 不干预具体实现 | **完全不干预** |
| 命名规范 | PMO 规范 (Requirement-Engineer 等) | 业务项目自定 (Data-Engineer 等) |
| 数量 | 5 个 (或简化 2-3 个) | 业务项目自定 |
| 元规则 | 0.0.11 + 0.0.13 | **0.0.14 (本规则)** |

**关键**: PMO 监督**只看 5 阶段 agent 上报数据**, **不监督业务 agent**。

## 业务项目 m2.6 7 项合规 (PMO 监管)

业务项目**整体**作为 1 个单位, 必须符合 PMO m2.6 7 项合规:

1. 业务项目注册 (m2.6)
2. 业务项目状态机 (active/paused/blocked/completed/archived)
3. 业务项目配额 4 维 (Token/时间/存储/并发)
4. 业务项目归档 4 层面 (数据/文档/项目/资源)
5. 业务项目隔离 3 维 (数据/配置/状态)
6. 业务项目告警 3 层 (业务自给/PMO/Sponsor)
7. 业务项目 checklist 6+3 项

**注意**: m2.6 7 项合规是业务项目**整体**合规, 监管业务项目本身, **不监管业务 agent 内容**。

**业务项目 m2.6 7 项合规 vs 5 阶段 PMO 7 项合规**:
- m2.6 7 项: 业务项目**整体** (m2.6 是 PMO 模块, 监管业务项目本身)
- 5 阶段 PMO 7 项: 业务项目**5 阶段 agent** (0.0.13, 监管 5 阶段 agent)
- 业务 agent: **不监管** (0.0.14, 业务自管)

## 业务项目内 2 类 agent 边界总结

```
业务项目 (1 个单位)
├── eng-roles/              ← 5 阶段研发 agent (PMO 监管, 0.0.13)
│   ├── 01-requirement-engineer.py
│   ├── 02-development-engineer.py
│   ├── 03-test-engineer.py
│   ├── 04-operations-engineer.py
│   └── 05-evaluation-engineer.py
├── biz-agents/             ← 业务需求运营 agent (业务自管, 0.0.14)
│   ├── 01-data-engineer.py
│   ├── 02-quant-analyst.py
│   └── ... (业务项目自定)
├── reports/                ← 5 阶段上报 (PMO 监督源)
└── biz-data/               ← 业务数据 (业务自存, PMO 不存)
```

**PMO 监督 = 看 eng-roles/ 上报数据**
**PMO 不监督 = biz-agents/ 内容 + biz-data/ 业务数据**
