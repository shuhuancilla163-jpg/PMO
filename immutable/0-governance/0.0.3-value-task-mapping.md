# 0.0.3 价值 → 任务映射 (Sponsor 定的)

每个任务必须能回答"我服务于哪个价值?", 否则不接。

## 价值-任务映射

| 价值 | 服务任务 |
|---|---|
| **高效流程** | 阶段门控 (C1/C2/C3), 模板化, 触点抽象 (8.0-8.5) |
| **规范** | 不可变文档 (F1/F5), 接口契约 (H-1), 模板 |
| **可控** | 自检 (D1-D16), 性能管理, 异常拦截 |
| **验收** | 阶段验收, 决策日志, MVP 验收 |
| **优化** | Memory Manager, 反思, 规则演进 |
| **工程化** | Token 追踪, Memory 分级, 触发激活, 上下文压缩, 触点抽象 |

## 价值-任务关系链 (示例)

### 示例 1: 阶段门控
- 价值: 高效流程
- 任务: m1.3 核心执行
- 落地: 阶段门控 (P0-P5), 模板化, 触点抽象
- 验收: 7 项核心执行可跑

### 示例 2: 不可变文档
- 价值: 规范
- 任务: m0.1 存储层
- 落地: Git 不可变文档库 (immutable/)
- 验收: 不可变文档可写入 + tag/release

### 示例 3: 自检
- 价值: 可控
- 任务: m1.5 自检
- 落地: 9 项自检 (D1-D16) + PMO 升级机制 + Sponsor 报告
- 验收: 9 项自检可跑

### 示例 4: 阶段验收
- 价值: 验收
- 任务: m1.3 核心执行
- 落地: 阶段验收 (P0-P5) + 决策日志
- 验收: 阶段验收节点清晰

### 示例 5: Memory Manager
- 价值: 优化
- 任务: m0.1 存储层 + m1.3 核心执行
- 落地: Memory 分级 (热/温/冷) + 反思 (0.0.8 自进化)
- 验收: Memory 可用 + 反思可工作

### 示例 6: Token 追踪
- 价值: 工程化
- 任务: m1.3 核心执行
- 落地: Token 追踪 + 预算分配
- 验收: Token 可追溯

## DEC-2026-0002 调整 (Sponsor 平等讨论后)

### 价值-任务映射扩展 (3 维度)

| 价值 | 服务任务 (3 维度) |
|---|---|
| **高效流程** | 8 PMO 角色 (按 3 维度分离) + 5 阶段研发角色 + 业务项目接入 5 步流程 |
| **规范** | 不可变文档 + 接口契约 + 业务项目 2 层合规契约 (业务项目整体 + 研发 5 阶段) |
| **可控** | 3 维度分别考核 (Assessor-Agent) + 3 维度监控 (PMO-Main/Engineer-Agent/Monitor-Agent) + 3 层上报机制 |
| **验收** | 阶段验收 + 决策日志 + 3 维度考核 (业务项目整体 / 研发 5 阶段 / 业务项目上报) |
| **优化** | Memory Manager + 反思 (0.0.8 自进化) + 业务自净化 |
| **工程化** | Token 追踪 + Memory 分级 + 触发激活 + 上下文压缩 + 项目间消息经 PMO 中介 |

## 关键 (Sponsor 定的)

- 每个任务必须有价值归属
- 每个价值必须有具体任务落地
- 价值-任务关系链可追溯
- DEC-2026-0002 后, 价值-任务关系链扩展到 3 维度

**不可变 (Sponsor 定的价值-任务映射)**
