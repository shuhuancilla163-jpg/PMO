# 0.0.4 价值 → 失败检测 (Sponsor 定的)

价值失效 = 价值没落地。每个价值必须有失效检测 + 修复方案。

## 价值失效检测

| 价值失效 | 表现 | 修复 |
|---|---|---|
| **高效流程失效** | 流程卡住, 反复重做 | 重新设计阶段门控 (P0-P5) + 模板化 |
| **规范失效** | agent 行为不一致, 输出格式乱 | 强化不可变文档 + 模板 + 接口契约 |
| **可控失效** | 不知道当前状态, 风险不可见 | 加强自检 + 状态报告 + 3 维度监控 |
| **验收失效** | Sponsor 不清楚拍板时机 | 强化阶段验收节点 + Sponsor review SOP |
| **优化失效** | 重复犯错, 规则不演进 | 强化 Memory Manager + 反思 (0.0.8) |
| **工程化失效** | Token 暴涨, 上下文超限 | 强化 Memory 分级 + 触发激活 + 上下文压缩 |

## 失效检测机制 (m1.5 9 项自检)

| 价值 | 自检 (D 项) |
|---|---|
| **高效流程** | D1 阶段流转合规 + D2 阶段门控生效 |
| **规范** | D3 不可变文档完整性 + D4 接口契约一致性 |
| **可控** | D10 跨级汇报检测 + D11 主 agent 越权检测 + D12 子 agent 决策越权检测 + D13 异常拦截检测 + D16 指标可贯彻检测 |
| **验收** | D5 决策日志完整性 + D6 阶段验收节点 |
| **优化** | D7 Memory 分级正确性 + D8 反思日志 |
| **工程化** | D9 Token 预算 + D14 上下文管理 + D15 触发器激活 |

## 价值失效 4 步修复流程

1. **检测**: 9 项自检 (m1.5) 触发失效信号
2. **报告**: 自动报告 Sponsor (m0.4 告警 3 层)
3. **诊断**: Engineer-Agent (m1.4) 诊断失效根因
4. **修复**: PMO-Main (m1.4) 触发修复 + Sponsor review

## DEC-2026-0002 调整 (Sponsor 平等讨论后)

### 价值失效检测扩展 (3 维度)

| 价值失效 (3 维度) | 表现 | 修复 |
|---|---|---|
| **高效流程失效 (维度 1)** | 业务项目注册/状态/配额/归档/隔离异常 | PMO-Main 修复 + Assessor 考核 |
| **高效流程失效 (维度 2)** | 业务项目内研发 5 阶段不符合 PMO 7 项 | Engineer-Agent 修复 + Assessor 考核 |
| **高效流程失效 (维度 3)** | 业务项目上报不合规 | Monitor-Agent 监管 + Assessor 考核 |
| **可控失效 (3 维度)** | 3 维度分别不可控 | 3 维度监控 (PMO-Main / Engineer-Agent / Monitor-Agent) |
| **验收失效 (3 维度)** | Sponsor 不清楚 3 维度拍板时机 | 3 维度分别考核 (Assessor-Agent) + Sponsor review SOP |

### 3 维度失效检测机制 (新)

| 维度 | 检测 |
|---|---|
| **维度 1: 业务项目整体** | PMO-Main 实时监控 (m1.4 维度 1 采集) + Assessor 考核 |
| **维度 2: 业务项目内研发 5 阶段** | Engineer-Agent 实时监控 (m1.4 维度 2 采集) + Assessor 考核 |
| **维度 3: 业务项目内业务上报** | Monitor-Agent 实时监控 (m1.4 维度 3 采集) + Assessor 考核 |

### 3 维度失效自动上报 (新)

- 维度 1 失效 → 业务项目整体告警 (L1 PMO-Main → L0 Sponsor)
- 维度 2 失效 → 业务项目内研发 5 阶段告警 (L2 Engineer-Agent → L1 PMO-Main)
- 维度 3 失效 → 业务项目上报合规告警 (L2 Monitor-Agent → L1 PMO-Main)

## 关键 (Sponsor 定的)

- 每个价值必须有失效检测
- 失效检测有修复方案
- 失效检测可自动触发
- DEC-2026-0002 后, 失效检测扩展到 3 维度

**不可变 (Sponsor 定的价值-失败检测)**
