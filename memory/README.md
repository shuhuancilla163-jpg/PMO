# Memory 分级 (0.0.2 工程化)

PMO Memory 按访问频率和重要性分 3 级, 平衡 **性能** (热数据快速) 和 **持久** (冷数据完整)。

## 3 级

| 级别 | 路径 | 访问频率 | 持久化 | 用途 |
|---|---|---|---|---|
| **热 (hot)** | `memory/hot/` | 实时 | 内存 + 临时文件 (`.gitignore`) | 当前活跃状态, agent 当前任务 |
| **温 (warm)** | `memory/warm/` | 近期 | Git 跟踪 | 近期决策, 近期任务, 近期指标 |
| **冷 (cold)** | `memory/cold/` | 历史 | Git 跟踪 | 历史决策, 历史任务, 历史指标, 复盘报告 |

## 写入规则

- **热 → 温**: agent 任务完成时, 把热数据归档到温
- **温 → 冷**: 项目阶段完成时, 把温数据归档到冷
- **冷 → 决策日志**: 重大决策永久写入 `decisions/active/`

## Token 经济

- **热数据**: 加载到 LLM context, 占用 Token 多但访问快
- **温数据**: 按需加载 (触发式), 占用 Token 少
- **冷数据**: 不加载到 LLM, 检索时按需拉取

## 与 0.0.8 自进化关系

- 温数据是自进化的**主要数据源** (近期反思)
- 冷数据是自进化的**历史数据源** (复盘)
- 热数据是自进化的**实时数据源** (实时反馈)
