# PMO 7 项合规检查模板 (业务项目接入时用)

**业务项目接入 PMO 时, 必须通过本检查**

## PMO 7 项合规检查项

业务项目 5 阶段每个阶段都必须通过以下 7 项检查:

### 1. 阶段门控 (m1.3)
- [ ] 阶段有进入门控
- [ ] 阶段有退出门控
- [ ] 阶段有验收标准
- [ ] 门控通过证据记录

### 2. 决策日志 (m1.1)
- [ ] 阶段决策有记录
- [ ] 决策有理由
- [ ] 决策有风险评估
- [ ] 决策有 Sponsor 签

### 3. 不可变文档 (m0.1)
- [ ] 阶段产出有不可变文档
- [ ] 文档有签名
- [ ] 文档有版本号
- [ ] 文档有归档

### 4. 异常拦截 (m0.4)
- [ ] 阶段异常有拦截
- [ ] 异常有分类 (L1/L2/L3)
- [ ] 异常有处理
- [ ] 异常有上报

### 5. 指标可贯彻 (m1.2)
- [ ] 阶段指标有定义
- [ ] 指标可测量
- [ ] 指标可追溯
- [ ] 指标可审计

### 6. 3 层告警 (m0.4)
- [ ] 业务自给告警 (业务项目内)
- [ ] PMO 告警 (L1 PMO-Main)
- [ ] Sponsor 告警 (L0 Sponsor)

### 7. Sponsor 报告 (m1.5)
- [ ] 阶段给 Sponsor 报告
- [ ] 报告有阶段进度
- [ ] 报告有风险预警
- [ ] 报告有决策请求

## PMO 监督 (L2 PMO-Engineer-Agent 采)

PMO Engineer-Agent 监督:
- 业务项目 5 阶段**上报数据** 是否符合 7 项
- 业务项目 5 阶段**决策日志** 完整性
- 业务项目 5 阶段**不可变文档** 数量
- 业务项目 5 阶段**异常拦截率**
- 业务项目 5 阶段**指标上报率**

**PMO 不监督**:
- 业务项目 5 阶段 agent **具体实现代码**
- 业务项目 5 阶段 agent **业务特定工具**
- 业务项目 5 阶段 agent **业务特定输入输出字段**

## 违反处理

| 违反 | 处理 |
|---|---|
| 必选阶段未实施 | critical + 业务项目暂停 |
| 阶段不符合 PMO 7 项 | 警告 + 业务项目整改 |
| 阶段未走 PMO 阶段门控 | critical + 业务项目暂停 |
| 阶段未记录决策日志 | 警告 + 补录决策日志 |
| 阶段不可变文档缺失 | 警告 + 补录不可变文档 |
| 阶段异常拦截失败 | critical + 升级到 Sponsor |
| 阶段指标未上报 | 警告 + 补报指标 |
| 阶段告警失败 | 告警通道修复 + 重新告警 |
| 阶段未给 Sponsor 报告 | 警告 + 补 Sponsor 报告 |

## 关键

- **PMO 7 项合规不可减少** (业务项目可增加)
- **必选阶段 (研发/测试/运维) 不可关闭**
- **业务项目自评 + PMO 监督**
- **违反处理 critical + 警告**
